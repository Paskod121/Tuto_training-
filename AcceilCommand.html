Commandes de base

git init

Initialise un nouveau dépôt Git dans le répertoire courant, créant une structure pour suivre les modifications de vos fichiers.

Voir en détail

git clone

Copie un dépôt existant vers un nouveau répertoire local, permettant de travailler sur un projet déjà existant.

Voir en détail

git add

Ajoute des fichiers à la zone de staging en préparation pour un commit, permettant de sélectionner ce qui sera enregistré.

Voir en détail

git commit

Enregistre les changements dans l'historique du dépôt avec un message descriptif expliquant les modifications effectuées.

Voir en détail

Travail avec les branches

git branch

Liste, crée ou supprime des branches dans le dépôt, permettant de travailler sur différentes fonctionnalités en parallèle.

Voir en détail

git checkout

Bascule entre les branches ou restaure des fichiers, permettant de naviguer dans l'historique du projet.

Voir en détail

git merge

Fusionne une branche dans la branche courante, intégrant les modifications d'une branche de fonctionnalité dans la branche principale.

Voir en détail

git rebase

Réapplique les commits d'une branche sur une autre, permettant de maintenir un historique plus linéaire du projet.

Voir en détail

Commandes de collaboration

git pull

Récupère et intègre les changements depuis un dépôt distant, combinant fetch et merge en une seule commande.

Voir en détail

git push

Envoie les changements locaux vers un dépôt distant, partageant vos modifications avec d'autres collaborateurs.

Voir en détail

git fetch

Récupère les changements depuis un dépôt distant sans les fusionner, permettant d'examiner les modifications avant intégration.

Voir en détail

git remote

Gère les dépôts distants associés à votre dépôt local, permettant d'ajouter, supprimer ou modifier les connexions avec d'autres dépôts.

Voir en détail
